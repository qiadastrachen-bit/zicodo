### 2. 游戏状态保持
- **需求**：用户离开互动页再返回，游戏进度必须保持
- **实现方案**：使用 Vue 的 `<keep-alive>` 组件缓存
- **配置**：
  ```vue
  <router-view v-slot="{ Component }">
    <keep-alive :include="['InteractPage', 'PetHomePage']">
      <component :is="Component" />
    </keep-alive>
  </router-view>
  ```
- **组件名设置**：
  ```js
  defineOptions({
    name: 'InteractPage'  // 必须设置，否则 keep-alive 不生效
  })
  ```
- **生命周期**：
  - 切换到其他页面：`onDeactivated` 触发
  - 返回互动页：`onActivated` 触发
  - 可选：暂停/恢复计时器
- **状态**：✅ 已确认，不允许再次变化

## 技术架构（已确认）

### 组件结构
```
src/pages/interact/
├── InteractPage.vue          ← 主页面（路由 /interact）
src/components/
├── game/
│   ├── GameBoard.vue        ← 8×8 游戏网格
│   ├── GameCell.vue         ← 单个格子（汉字+动效）
│   ├── GameScore.vue        ← 分数 + 时间显示
│   └── GameComplete.vue     ← 通关弹窗（可选）
├── common/
│   └── ZlTabbar.vue        ← TabBar 组件（增量修改）
src/game/
├── chars.js                 ← 字库（已创建）
└── game-logic.js            ← 游戏核心逻辑（状态管理）
src/styles/
└── game.css                 ← 游戏专属样式（动效）
```

### 数据流
1. **字库**：`src/game/chars.js` → `CHAR_POOL`（166 字）
2. **游戏状态**：`GameLogic` 类（本地 reactive）→ `score` / `time` / `cells`
3. **通关判定**：本地计算（全部 `cleared === true`）
4. **零 API**：游戏逻辑完全本地，不调用任何后端接口

### 动画实现
1. **墨色溢出（ink-burst）**：消除时，CSS animation（300ms）
2. **抖动（shake）**：判错时，CSS animation（200ms）
3. **新字填入（refill）**：消除后补字，CSS animation（200ms）
4. **TabBar 切换动画**：方案 B（scale + 颜色变化）
