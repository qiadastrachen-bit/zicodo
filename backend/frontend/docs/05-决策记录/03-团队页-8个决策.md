### 5. 轮播文案功能（2026-06-08 04:53 新增）
- **需求**：文案可以自定义、可以轮播、自由变换
- **实现方式**：
  - `LoadingState.vue` 支持 `messages` props（数组，支持多个文案）
  - 支持 `interval` props（轮播间隔，默认 3000ms）
  - 组件内部自动轮播（onMounted 设置定时器）
  - 组件销毁时自动清除定时器（onUnmounted）
- **使用示例**：
  - 单个文案：`<LoadingState message="加载中..." />`
  - 多个文案：`<LoadingState :messages="['功能正在上线', '敬请期待']" :interval="3000" />`
  - 自由变换：动态修改 `messages` 数组，文案自动更新
- **默认轮播文案**（功能正在上线/正在设计）：
  ```js
  const defaultMessages = ['功能正在上线', '敬请期待', '即将开放']
  ```
- **状态**：✅ 已确认（2026-06-08 04:53）

## 技术架构

### 数据流
```
用户操作 → Teams Store (Pinia) → localStorage
                           ↓
                     TeamCreateForm → 新建团队
                           ↓
                     TeamDetail → 查看详情
                           ↓
                     TeamTaskPage → 团队任务（复用 TaskCard）
                           ↓
                     syncTaskToPersonal() → 同步到个人任务
```

### 关键文件
| 文件 | 说明 |
|------|------|
| `src/stores/teams.js` | 团队数据读写 + 成员管理 |
| `src/components/schedule/TeamCard.vue` | 团队卡片 |
| `src/components/schedule/TeamCreateForm.vue` | 新建团队弹窗 |
| `src/components/schedule/TeamDetail.vue` | 团队详情（任务+成员） |
| `src/components/schedule/TeamTaskPage.vue` | 团队任务列表（复用） |
| `src/components/schedule/MemberList.vue` | 成员详情页（姓名+角色） |
| `src/components/schedule/EmptyState.vue` | 通用空状态 |

### 默认数据
- 初始化时注入 `DEFAULT_TEAMS`（包含"王之故乡"团队 + 5人分工）
- 用户可以在此基础上修改或新建团队

## 交互设计

### 团队列表视图
- **有团队**：显示 TeamCard 列表（名称、人数、菜单按钮）
- **无团队**：显示 EmptyState（颜文字 + "您还没有加入团队"）

### 团队详情视图
- **任务卡片**：点击进入团队任务列表（复用个人任务的交互）
- **成员列表**：点击进入成员详情页（显示姓名 + 角色分工）

### 团队任务交互
- **打卡**：复用 CheckInTimer.vue（与个人任务完全一致）
- **删除**：复用 TaskCard.vue 的删除逻辑
- **同步到个人**：任务卡片添加"同步"按钮 → 调用 `syncTaskToPersonal()`

### 新建团队
- **触发**：点击团队列表右上角 + 按钮
- **表单**：名称（必填）、备注（可选）、座右铭（可选）
- **确认**：创建团队 → 添加到列表 → 关闭弹窗
