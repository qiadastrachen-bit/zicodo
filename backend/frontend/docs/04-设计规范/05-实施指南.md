## 📂 文件清单

我已经为你创建了 3 个提示词文件，用于指导执行机器人实施设计规范：

### 1. `设计规范-更新tokens.md`
- **用途**：指导执行机器人更新 `tokens.css` 文件
- **优先级**：最高（所有组件和页面依赖此文件）
- **预计用时**：30-45 分钟
- **输出文件**：`src/styles/tokens.css`

### 2. `设计规范-组件示例.md`
- **用途**：指导执行机器人创建/重写 5 个核心组件
- **优先级**：高（为所有页面提供统一的组件基础）
- **预计用时**：1-1.5 小时
- **输出文件**：`src/components/Button.vue`、`Input.vue`、`Card.vue`、`Modal.vue`、`EmptyState.vue`

### 3. `设计规范-设计审查.md`
- **用途**：指导执行机器人审查现有页面是否符合设计规范
- **优先级**：中（确保所有页面符合规范）
- **预计用时**：45-60 分钟
- **输出文件**：`design-review-report.md`

## 🔧 使用流程

### Step 1： read 设计规范文件

**在开始之前，你必须先阅读设计规范**，确保你理解所有 Token 的含义：

```
文件路径：G:\陈锦彤2026上\智能开源硬件\前端\design-system-spec.md
```

**重点关注**：
- 颜色系统（13 个 Token）
- 字体系统（7 级层次）
- 间距系统（8pt Grid）
- 组件尺寸规范（按钮、输入框、卡片等）

---

### Step 2：开新对话，发送第一个提示词

**操作流程**：
1. 打开 WorkBuddy，点击 "+ 新对话"
2. 等待执行机器人回复（"你好，我是执行机器人，请发送任务提示词"）
3. 打开 `设计规范-更新tokens.md` 文件
4. 复制全部内容，粘贴到新对话中
5. 发送！

**重要提醒**：
- 必须使用**新对话**（不要在顾问对话中发送）
- 必须等待执行机器人回复后，再发送提示词
- 必须复制**全部内容**（不要遗漏任何部分）

---

### Step 3：等待执行机器人完成

**执行机器人工作流程**：
1. 读取提示词
2. 检查 `tokens.css` 是否已存在
3. 创建/更新 `tokens.css` 文件
4. 更新 `global.css` 和 `main.js`
5. 测试 Token 是否可用
6. 生成交付报告

**预计用时**：30-45 分钟

**你需要做的**：
- 等待执行机器人完成
- **不要中途打断**
- 如果超过 1 小时没回复，可以提醒一下

---

### Step 4：验收执行机器人交付

**执行机器人完成后，会提供**：
1. **文件目录结构**（确认文件位置正确）
2. **Token 清单**（确认所有 Token 都已定义）
3. **测试证明**（浏览器截图，显示 Token 正常工作）
4. **遵守规则确认**（勾选所有规则）

**你的验收步骤**：
1. 检查文件是否真的存在（去 `src/styles/` 目录查看）
2. 检查 `tokens.css` 内容是否完整（对比提示词中的清单）
3. 启动开发服务器（`npm run dev`）
4. 打开浏览器，访问 `http://localhost:5173`
5. 按 F12 打开开发者工具，检查元素样式是否使用了 Token

**验收通过标准**：
- 文件存在，内容完整
- Token 可以在 Vue 组件中使用（`var(--zl-brand)`）
- 浏览器中显示正确的样式

**验收不通过**：
- 如果文件不存在，或者内容不完整 → 要求执行机器人重新执行
- 如果 Token 不可用 → 要求执行机器人检查 `global.css` 和 `main.js` 的引入

---

### Step 5：继续下一个提示词

**第一个提示词验收通过后**，继续第二个：

1. 打开**新的对话**（不要使用之前的对话）
2. 等待执行机器人回复
3. 打开 `设计规范-组件示例.md` 文件
4. 复制全部内容，粘贴到新对话中
5. 发送！

**重复 Step 3 和 Step 4**，直到所有 3 个提示词都执行完成。

## 🚨 常见问题

### Q1：执行机器人没有回复怎么办？

**A1**：
- 等待 5-10 分钟（执行机器人可能在思考）
- 如果超过 10 分钟没回复，可以发送 "你还在吗？"
- 如果超过 30 分钟没回复，可以截图发给我（顾问对话），我帮你分析

### Q2：执行机器人回复"我不理解这个任务"怎么办？

**A2**：
- 截图发给我（顾问对话），我帮你分析
- 可能是提示词不够清晰，我会帮你修改提示词
- 不要自己随意修改提示词（可能会导致更多问题）

### Q3：执行机器人交付的文件内容不完整怎么办？

**A3**：
- 不要验收通过！
- 回复执行机器人："文件内容不完整，缺少 XXX Token，请重新检查"
- 等待执行机器人重新交付

### Q4：我不知道如何验收怎么办？

**A4**：
- 截图发给我（顾问对话），我帮你验收
- 或者，你可以把所有交付内容（文件目录、Token 清单、测试截图）截图发给我，我帮你检查
