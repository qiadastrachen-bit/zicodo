## 总则：好提示词的三个标准

### 标准一：执行者不需要猜测任何事

❌ 坏：「点击日期后显示当天的任务」  
✅ 好：「点击日期后 → 下方待办区用 `scrollIntoView({ behavior: 'smooth' })` 平滑滚动到视口内 → 显示该日期的任务列表」

**规则**：每个交互都要写清楚「触发 → 行为 → 结果」三步。

### 标准二：覆盖四种状态

每个 UI 区域都要写明四个状态的表现：

| 状态 | 必须说明 | 反面案例 |
|------|---------|---------|
| **默认态** | 正常加载后的样子 | 没说，执行者就会省略 |
| **空状态** | 没有数据时的样子 + 文案 | 写了"列表渲染"但没说空的时候怎么办 |
| **加载态** | 数据加载中的样子（骨架屏/loading图/spinner） | 没说，执行者就不加 loading |
| **错误态** | 接口失败/数据异常的样子 + 重试按钮 | 没说，执行者只写 try-catch 不写 UI |

### 标准三：禁止歧义，一切量化

❌ 坏：「卡片之间有一些间距」  
✅ 好：「卡片间距 `var(--zl-space-3)` = 12px」

❌ 坏：「弹窗从底部弹出」  
✅ 好：「弹窗使用 `transform: translateY(100%)` → `translateY(0)`，动画 `var(--zl-transition-normal)` 300ms」

### 组件4：TaskList.vue

| 维度 | 规格 |
|------|------|
| **路径** | `src/components/calendar/TaskList.vue` |
| **Props** | `tasks` (Array), `selectedDate` (String, nullable) |
| **Emits** | `toggle-complete(taskId)`, `open-detail(taskId)` |

**四种状态**：

| 状态 | 触发 | UI |
|------|------|-----|
| **默认态** | 选中日期，有任务 | 任务卡片列表，每张显示 ☐ 标题 + 时间段 |
| **空状态（选中但无任务）** | selectedDate 非空，tasks 为空 | `<EmptyState>` 组件，文案："这天没有任务，添加一个吧 ✨" |
| **空状态（未选中）** | selectedDate 为 null | `<EmptyState>` 组件，文案："选择一个日期查看任务" |
| **加载态** | — | 本页数据来自 localStorage，无需加载态 |

**交互行为**：

| 操作 | 行为 |
|------|------|
| 点击复选框 | toggle 任务完成状态 → 立即更新 `scheduleStore` → ⚠️ **同时更新 CalendarGrid 的任务标记点** |
| 点击任务卡片 | emit `open-detail(taskId)` → 父组件打开 TaskDetailSheet |
| 点击 [+添加] | 跳转 `/add` 页面（新建任务） |

**⚠️ 数据联动规则（最容易漏！）**：
```js
// 在 toggleComplete() 中，必须立即同步更新日历标记点
function toggleComplete(taskId) {
  const task = tasks.value.find(t => t.id === taskId)
  if (task) {
    task.completed = !task.completed
    saveToStorage()  // 持久化到 localStorage
    
    // ⚠️ 关键：触发日历网格重新计算该日期的标记点
    // 方式：更新 scheduleStore 中的 tasks，CalendarGrid 的 computed 会自动响应
    scheduleStore.refreshTasks() 
  }
}
```

**无障碍**：
- 任务列表 `role="list"`
- 每个任务卡片 `role="listitem"`
- 已完成的复选框 `aria-checked="true"`

## 🔄 数据流

### 数据来源

| 数据 | 来源 | 存储位置 | 更新时机 |
|------|------|---------|---------|
| 所有任务 | `scheduleStore.tasks` | localStorage `ziling_schedule_tasks` | 页面 mount 时读取 |
| 当前月份 | 组件内 `ref` | 仅内存 | 月份切换时 |
| 选中日期 | 组件内 `ref` | 仅内存 | 点击日期时 |

### 数据联动规则（⚠️ 重点！）

| 操作 | 影响 | 需要同步更新的 UI |
|------|------|------------------|
| toggle 任务完成 | `task.completed` 翻转 | ⚠️ CalendarGrid 标记点立即更新（消失/变灰） |
| 删除任务 | 任务从数组中移除 | CalendarGrid 标记点 + TaskList 列表 |
| 切换月份 | `currentMonth` ref 更新 | CalendarGrid 重新渲染 → 选中状态重置 |
| 新建任务（从 + 按钮） | 跳转 `/add` 页 | 返回日历页时，重新读取 localStorage |

## 📱 边界情况

### 最小屏幕 (320px)
- 日期格子：`min(44px, (100vw - 32px) / 7)` — 确保7列能放下
- 如果 `(320 - 32) / 7 ≈ 41px < 44px`：格子缩小到 41px（可接受的降级）

### 内容溢出
- 任务标题过长（>15字）：`text-overflow: ellipsis; white-space: nowrap;`
- 任务列表超出屏幕：在 TaskList 区域内部滚动，不影响日历网格
- 月份切换：日历网格高度变化时，页面平滑过渡

### 边界情况处理
- 首次使用（localStorage 无数据）：显示空日历 + "还没有任务，去添加一个吧 ✨" 引导
- 12月 → 1月切换：年份自动 +1
- 1月 → 12月切换：年份自动 -1
- 用户快速连点日期：防抖处理，300ms 内不重复触发

## 常见翻车场景 & 补救话术

### 翻车1：执行者写的组件没空状态

**现象**：数据为空时页面白屏或报错。

**原因**：提示词只写了"用 v-for 渲染列表"，没说没数据时怎么办。

**补救话术**（下次提示词加上）：
> ⚠️ 补一个要求：当 `tasks` 数组为空时，不要只渲染空列表。使用 `EmptyState.vue` 通用组件，文案为"【具体文案】"。空状态也要居中显示，和列表区域等高等宽。

---

### 翻车2：执行者硬编码了颜色/字号

**现象**：审查时发现 `color: #87C8B4` 或 `font-size: 16px`。

**原因**：提示词自己写了硬编码值作为"示例"。

**补救**：
- 提示词中**永远不要出现任何硬编码色值作为示例**。
- 示例代码中也要用 `var(--zl-xxx)`。
- 例外：白色 `#FFFFFF` 和黑色 `#000000` 可以在「今天日期文字色」这类特例中标注，但必须先在 Token 中定义。

---

### 翻车3：弹窗突然出现/消失

**现象**：弹窗没有过渡动画，"啪"一下出现。

**原因**：提示词没写动画规格。

**补救**（每次提到弹窗都加这句）：
> 弹窗打开：遮罩 `opacity 0→1` + 内容 `translateY(100%)→0`，动画 `var(--zl-transition-normal)` 300ms。关闭反向。

---

### 翻车4：组件放错目录

**现象**：团队组件放在了 `components/schedule/` 下，通用组件散落各处。

**原因**：提示词没写清楚文件路径，执行者自己猜。

**补救**：每个提示词开头必须有一个「🗂️ 文件路径」表格，每行写明组件的完整路径。

---

### 翻车5：数据更新后 UI 没联动

**现象**：用户完成任务后，日历上的标记点还在。

**原因**：提示词只写了"完成任务 = 改 completed 字段"，没写"改了之后还要通知日历组件重算标记点"。

**补救**：每个数据修改操作后面都要加一行"⚠️ 数据联动"说明。
