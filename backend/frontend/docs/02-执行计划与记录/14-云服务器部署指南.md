# 云服务器部署指南

**适用系统**: Ubuntu 20.04+ / CentOS 7+ / Debian 10+（推荐 Ubuntu 22.04 LTS）  
**数据库**: SQLite + umzug 迁移系统  
**进程管理**: PM2  
**反向代理**: Nginx（可选，建议配置 HTTPS）

---

## 一、服务器准备

### 1.1 基础环境安装

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 20.x（LTS）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 验证版本
node -v  # 应输出 v20.x.x
npm -v   # 应输出 10.x.x

# 安装 PM2（进程管理器）
sudo npm install -g pm2

# 安装 Git
sudo apt install -y git
```

### 1.2 创建项目目录和用户

```bash
# 创建专用运行用户（可选，推荐）
sudo useradd -m -s /bin/bash digital-life
sudo usermod -aG sudo digital-life

# 或直接用当前用户操作
mkdir -p /opt/digital-life
cd /opt/digital-life
```

## 二、部署项目

### 2.1 拉取代码

```bash
cd /opt
git clone <your-repo-url> digital-life
cd /opt/digital-life

# 切换到生产分支
git checkout main
```

### 2.2 安装依赖

```bash
cd /opt/digital-life/backend/ziling-backend
npm ci --production
# 或
npm install
```

### 2.3 配置环境变量

```bash
cd /opt/digital-life/backend/ziling-backend
cp .env.example .env
nano .env

# 主要配置项：
# PORT=3000
# DB_PATH=./data/digital_life.db
# JWT_SECRET=<生产环境请换成复杂随机字符串>
# NODE_ENV=production
```

> ⚠️ **重要**: `JWT_SECRET` 在生产环境必须是足够复杂的随机字符串，绝不能用默认值。可以用以下命令生成：
> ```bash
> openssl rand -hex 64
> ```

### 2.4 初始化数据库（自动执行 migrations）

```bash
# 确保 data 目录存在
mkdir -p /opt/digital-life/backend/ziling-backend/data

# 执行初始 migration（启动后端时会自动执行，也可以手动执行）
node src/db/migrate.js up

# 验证
node src/db/migrate.js executed
# 应输出 8 个 migration 已执行
```

## 三、用 PM2 启动后端

### 3.1 启动

```bash
cd /opt/digital-life/backend/ziling-backend

# 启动并命名
pm2 start src/app.js --name digital-life-backend

# 查看状态
pm2 status

# 查看日志
pm2 logs digital-life-backend

# 开机自启
pm2 startup systemd
# 执行后会输出一个命令，复制粘贴执行即可
pm2 save
```

### 3.2 PM2 常用命令

```bash
pm2 start   <name|id>   # 启动
pm2 stop    <name|id>   # 停止
pm2 restart <name|id>   # 重启
pm2 delete  <name|id>   # 删除
pm2 logs    <name|id>   # 查看日志
pm2 status              # 查看所有进程状态
pm2 monit               # 监控面板
```

## 四、用 Nginx 做反向代理（推荐）

### 4.1 安装 Nginx

```bash
sudo apt install -y nginx
```

### 4.2 配置文件

```bash
sudo nano /etc/nginx/sites-available/digital-life
```

内容：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 前端静态文件（如果前后端分开部署）
    # location / {
    #     root /opt/digital-life/frontend/dist;
    #     try_files $uri $uri/ /index.html;
    # }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/digital-life /etc/nginx/sites-enabled/
sudo nginx -t   # 测试配置
sudo systemctl restart nginx
```

### 4.3 配置 HTTPS（用 Certbot）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
# 按向导操作，选择自动重定向 HTTP → HTTPS
```

## 五、数据库自动备份

### 5.1 备份脚本

```bash
# 创建备份脚本目录
mkdir -p /opt/digital-life/backup
cd /opt/digital-life/backup
nano backup.sh
```

内容（见 `backup.sh`）：

```bash
#!/bin/bash
# 数字生命后端数据库自动备份脚本
# 用法: ./backup.sh  或  ./backup.sh custom-name

PROJECT_DIR="/opt/digital-life/backend/ziling-backend"
BACKUP_DIR="$PROJECT_DIR/data/backups"
DB_FILE="$PROJECT_DIR/data/digital_life.db"
LOG_FILE="$PROJECT_DIR/data/backups/backup.log"

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
CUSTOM_NAME=${1:-backup}
BACKUP_FILE="$BACKUP_DIR/digital_life_${TIMESTAMP}_${CUSTOM_NAME}.db"

mkdir -p "$BACKUP_DIR"

echo "[$(date)] 开始备份..." >> "$LOG_FILE"

if [ ! -f "$DB_FILE" ]; then
    echo "[$(date)] 错误: 数据库文件不存在: $DB_FILE" >> "$LOG_FILE"
    exit 1
fi

# SQLite 备份的标准做法：用 .backup 命令确保一致性
sqlite3 "$DB_FILE" ".backup '$BACKUP_FILE'"

if [ $? -eq 0 ]; then
    SIZE=$(du -h "$BACKUP_FILE" | cut -f1)
    echo "[$(date)] 备份成功: $BACKUP_FILE (大小: $SIZE)" >> "$LOG_FILE"

    # 压缩（节省空间）
    gzip -f "$BACKUP_FILE"

    # 保留最近 30 天的备份
    find "$BACKUP_DIR" -name "digital_life_*.db.gz" -type f -mtime +30 -delete

    echo "[$(date)] 备份完成" >> "$LOG_FILE"
else
    echo "[$(date)] 备份失败" >> "$LOG_FILE"
    exit 1
fi
```

给执行权限：

```bash
chmod +x backup.sh
```

### 5.2 配置定时任务（每日凌晨 3 点自动备份）

```bash
crontab -e
```

添加：

```cron
0 3 * * * /opt/digital-life/backup/backup.sh daily
```

## 六、常见运维命令

### 6.1 查看数据库表

```bash
cd /opt/digital-life/backend/ziling-backend/data
sqlite3 digital_life.db
# sqlite> .tables
# sqlite> SELECT * FROM users;
# sqlite> .exit
```

### 6.2 迁移相关

```bash
cd /opt/digital-life/backend/ziling-backend

# 查看已执行 migration
node src/db/migrate.js executed

# 查看待执行 migration
node src/db/migrate.js pending

# 执行所有待执行 migration
node src/db/migrate.js up

# 回滚最后一个 migration（慎重）
node src/db/migrate.js down
```

### 6.3 手动迁移（从老版本升级）

```bash
cd /opt/digital-life/backend/ziling-backend

# 1) 备份当前数据库
./backup.sh pre-migration

# 2) 拉取新版本代码
git pull origin main

# 3) 安装新依赖
npm install

# 4) 执行 migration（会自动执行新增的）
node src/db/migrate.js up

# 5) 重启后端
pm2 restart digital-life-backend
```

## 七、安全建议清单

| 项目 | 建议 |
|------|------|
| SSH | 禁用密码登录，用密钥；改端口号 |
| 数据库 | data/ 目录权限 700；数据库文件不放入 Git |
| 环境变量 | `.env` 文件权限 600；不在 Git 中 |
| HTTPS | 强制启用；HSTS 开启 |
| 防火墙 | 只开放 22, 80, 443 |
| 备份 | 每日自动备份 + 异地同步到 OSS/其他服务器 |
| 监控 | PM2 监控内存/CPU；Nginx access log 配合 fail2ban |

## 八、迁移系统生产环境操作注意事项

1. **执行 migration 前必须备份数据库**：`./backup.sh pre-migration`
2. **不直接在生产环境改 model.js**：任何数据库变更必须通过 migration 文件
3. **migration 文件一旦提交 Git，永不修改**：如果写错，写新的 migration 纠正
4. **执行 migration 期间关闭写入流量**（或至少保证只有一个节点执行 migration）
5. **大表变更（>10 万行）要评估锁表时间**，选择低峰期执行
6. **migration 执行后要跑冒烟测试**，验证核心功能正常
