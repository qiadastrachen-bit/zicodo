

`backend/.env` 示例：


## 五、域名与 HTTPS（推荐）

用户已确认有域名，建议与 IP 并行或替代 IP 作为对外推广链接。

### 5.1 DNS 解析

在 `hello.com` 的 DNS 控制台添加 **A 记录**：

| 主机记录 | 记录类型 | 记录值 | 说明 |
|----------|----------|--------|------|
| `zicodo` | A | `120.53.11.84` | 解析为 `zicodo.hello.com` |

验证（生效通常 5–30 分钟）：

```bash
ping zicodo.hello.com
# 或
nslookup zicodo.hello.com
```

### 5.2 SSL 证书（任选其一）

- **腾讯云**：轻量应用服务器 → 域名解析 → 申请免费 SSL  
- **Let's Encrypt**：服务器安装 certbot，配合 Nginx 443 端口  

启用 HTTPS 后：

1. 更新 `frontend/nginx.conf` 的 `server_name` 与 443 配置  
2. 在 `docker-compose.yml` 取消 `443:443` 注释并挂载证书目录  
3. 重新 `docker compose up -d --build`  
4. **重新生成二维码**，链接改为 `https://zicodo.hello.com`

## 十、风险与应对

| 风险 | 应对 |
|------|------|
| 80 端口无法访问 | 查 Lighthouse 防火墙 + `ss -tlnp \| grep 80` |
| 502 Bad Gateway | `docker compose logs backend`，查迁移与 `.env` |
| AI 对话失败 | 检查 `DEEPSEEK_API_KEY` |
| 微信内打开异常 | 提示「用系统浏览器打开」 |
| 数据丢失 | 勿 `docker compose down -v`；定期备份 db |
