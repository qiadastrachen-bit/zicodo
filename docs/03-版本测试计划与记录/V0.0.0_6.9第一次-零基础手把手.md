# zicodo 零基础手把手部署（第一轮内测）

> 写给完全没做过服务器部署的同学。按顺序做，做完一步打勾一步。  
> **当前推荐测试地址**：`http://120.53.11.84`（先用 IP，不依赖域名）

---

## 先搞懂三件事

### 1. 你的服务器是什么？

腾讯云轻量服务器，公网 IP：**120.53.11.84**  
上面已经装好 **Docker**，相当于一台 24 小时开着的远程电脑，用来跑网站。

### 2. 代码在哪里？

GitHub：https://github.com/qiadastrachen-bit/zicodo  
服务器会用 `git clone` 把代码下载下来，再用 Docker 启动。

### 3. 域名怎么回事？（重要）

你之前说的 **`zicodo.hello.com` 不能用**：

| 事实 | 说明 |
|------|------|
| `hello.com` | 是别人（Google）已经注册好的域名，**腾讯云不能帮你注册** |
| `zicodo.hello.com` | 网上查到的地址是 **Google 的服务器**，不是你的 `120.53.11.84` |
| 截图报错 | 正常，因为 `.hello.com` 这种后缀不在腾讯云可注册列表里 |

**第一轮内测不需要域名**，直接用：

```
http://120.53.11.84
```

手机扫码、浏览器打开都可以。以后想要「好记的网址 + HTTPS」，再在腾讯云买一个便宜域名（如 `zicodo.cloud`，轻量优惠约 1 元/首年）。

## 总流程一览

```
第 1 步  打开服务器防火墙 80 端口
第 2 步  用 SSH 登录服务器
第 3 步  在服务器下载代码
第 4 步  填写 backend/.env（密钥）
第 5 步  docker compose 启动
第 6 步  手机/电脑浏览器验证
第 7 步  生成二维码发给测试用户
```

预计耗时：**30～60 分钟**（第一次）

---

## 第 1 步：打开防火墙 80 端口

1. 打开 [腾讯云轻量控制台](https://console.cloud.tencent.com/lighthouse)
2. 点击你的实例 **Docker CE-mf9k**
3. 上方点 **「防火墙」**
4. 点 **「添加规则」**，填写：

| 应用类型 | 协议 | 端口 | 策略 | 来源 |
|----------|------|------|------|------|
| HTTP(80) | TCP | 80 | 允许 | 全部 IPv4 地址 |

5. 保存

> 22 端口一般默认已开（SSH 登录用）。第一轮 HTTP 只需 80。

---

## 第 2 步：SSH 登录服务器

### 2.1 获取登录密码

1. 在实例详情页，点 **「登录」** 或 **「重置密码」**
2. 用户名一般是 **`root`**
3. 设一个你能记住的密码（若从未设过）

### 2.2 在你电脑上打开 PowerShell

Win 键 → 输入 `PowerShell` → 回车

输入（把密码提示出来后输入，输入时屏幕不显示字符，正常）：

```powershell
ssh root@120.53.11.84
```

第一次会问 `Are you sure you want to continue connecting?`，输入 **`yes`** 回车。

看到类似 `root@VM-xxx:~#` 就表示**已经连上服务器**。

---

## 第 3 步：下载项目代码

在 **服务器** 的 SSH 窗口里，**整段复制粘贴**执行：

```bash
mkdir -p /opt/zicodo
cd /opt/zicodo
git clone https://github.com/qiadastrachen-bit/zicodo.git .
```

如果提示 `fatal: destination path '.' already exists`，说明以前克隆过，改用：

```bash
cd /opt/zicodo
git pull origin master
```

---

## 第 4 步：配置环境变量（最重要）

### 4.1 复制模板

在服务器执行：

```bash
cd /opt/zicodo
cp backend/.env.example backend/.env
nano backend/.env
```

### 4.2 用 nano 编辑（键盘操作）

你会看到几行配置，改成类似下面（**把尖括号换成真实值**）：

```env
DEEPSEEK_API_KEY=sk-你的DeepSeek密钥
AI_PROVIDER=deepseek
AI_MODEL=deepseek-chat
JWT_SECRET=随便写一串很长的随机字母数字
NODE_ENV=production
PORT=3000
DB_PATH=./data/zicodo.db
CORS_ORIGIN=*
```

| 配置项 | 怎么填 |
|--------|--------|
| `DEEPSEEK_API_KEY` | 在 https://platform.deepseek.com/ 申请，以 `sk-` 开头 |
| `JWT_SECRET` | 自己编 32 位以上随机字符，不要用示例里的默认值 |
| `CORS_ORIGIN` | 生产环境填 `*` 即可 |

**保存退出 nano**：

- `Ctrl + O` → 回车（保存）
- `Ctrl + X`（退出）

---

## 第 5 步：启动网站

在服务器执行：

```bash
cd /opt/zicodo
chmod +x scripts/server-first-deploy.sh
./scripts/server-first-deploy.sh
```

或手动执行：

```bash
cd /opt/zicodo
docker compose up -d --build
```

**第一次会等 5～15 分钟**（下载镜像、编译前端），属正常。

查看是否成功：

```bash
docker compose ps
```

两个容器都应是 `running` 或 `healthy`：

- `zicodo-frontend`
- `zicodo-backend`

再测健康接口：

```bash
curl http://localhost/api/health
```

应返回 JSON，里面有 `"status":"ok"` 或类似成功信息。

---

## 第 6 步：浏览器验证

### 在你电脑浏览器打开：

```
http://120.53.11.84
```

应看到 zicodo **开屏页** → 点进入 → **登录/注册一体页**。

### 建议自测清单

- [ ] 注册一个新账号
- [ ] 登录进入首页
- [ ] 创建一个团队，看到 6 位邀请码
- [ ] 个人中心 → 关于 zicodo 页面图标正常

### 如果打不开

| 现象 | 处理 |
|------|------|
| 无法连接 | 回到第 1 步检查防火墙 80 |
| 502 Bad Gateway | 服务器执行 `docker compose logs backend --tail 50` 看报错 |
| 页面白屏 | `docker compose logs frontend --tail 30` |

## 第 7 步：生成二维码（发给测试用户）

### 测试链接（复制这个）

```
http://120.53.11.84
```

### 生成方式（任选）

1. 打开 https://cli.im/ 或搜索「草料二维码」
2. 选择「网址」
3. 粘贴 `http://120.53.11.84`
4. 生成并下载图片

### 发给测试用户的话术

> 扫码打开 zicodo 内测版。  
> 1. 点「开始使用」注册（用户名 + 密码至少 6 位）  
> 2. 建议用 **Chrome / Safari** 打开，微信里可能功能不全  
> 3. 可「添加到主屏幕」当 App 用

## 以后：想要自己的域名（可选）

在腾讯云轻量控制台 **「域名」→ 注册新域名」**：

- 可买截图里的优惠后缀：`.cloud`、`.site`、`.icu` 等
- 例如注册 **`zicodo.cloud`**
- 解析：添加 A 记录 → 指向 `120.53.11.84`
- 再申请免费 SSL → 换成 `https://zicodo.cloud` 做二维码

**现在不必等域名，先用 IP 就能内测。**

## 常用运维命令（收藏）

```bash
cd /opt/zicodo

# 查看运行状态
docker compose ps

# 看日志
docker compose logs -f --tail=100

# 更新代码后重启
git pull origin master
docker compose up -d --build

# 备份数据库
docker cp zicodo-backend:/app/data/zicodo.db ./backup-$(date +%Y%m%d).db
```
