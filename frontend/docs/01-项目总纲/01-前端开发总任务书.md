# 字灵(ZiLing)前端开发 - 完整任务书

## 项目背景
字灵是一款数字汉字精灵宠物/AI伴侣APP。
- 后端已完成：Express.js + SQLite + DeepSeek AI
- 后端部署地址：**http://120.53.11.84**
- 项目状态：Figma设计稿已定，需要前端实现

## 技术栈（严格执行）
- 框架：Vue 3 (Composition API) + **JavaScript（不用TypeScript）**
- 构建工具：Vite + **npm**
- 路由：Vue Router 4
- 状态管理：Pinia
- 样式方案：CSS Variables（Design Tokens）+ 纯CSS（不用Tailwind）
- HTTP客户端：Axios（接口统一管理在 `src/api/` 目录）
- **不启用 ESLint**（个人项目，快速开发）
- 动画：**开屏动画使用已提供的splash包（public/splash/），游戏页保留反馈动效（消除/抖动/填入），其余页面全部取消动画，使用静态PNG图片**
- PWA：vite-plugin-pwa（后期再加）

## 开屏动画（Splash Screen — 已有完整包，直接接入）
**开屏动画由宠物负责同学交付，零第三方依赖，纯CSS+SVG实现。**
- **包位置：** `public/splash/` 目录（已放入）
- **时长：** ~5.45秒，分为4个阶段：
  1. **电路走线**（0~2s）：PCB线路从四周向中心绘制
  2. **Logo成型**（~1.2s）：走线汇聚成字灵Logo
  3. **眨眼**（~0.5s）：右眼眨一下
  4. **颜文字漂浮**（~1.7s）：^_^ Q_Q T_T 等颜文字四处飘散消失
- **颜色：** 背景 `#FFFAF1` = `--zl-bg` ✅ | Logo色 `#87C8B4` = `--zl-brand` ✅

### 接入方式（在 Vue 项目中使用）

**方法一：在 App.vue 中条件渲染（推荐）**
```vue
<!-- App.vue -->
<template>
  <!-- 开屏层：播放完自动移除 -->
  <div v-if="showSplash" class="splash-layer">
    <div data-ziling-splash data-auto-hide="true"
         @zilingSplashFinished="onSplashFinished"></div>
  </div>
  <!-- 主应用 -->
  <router-view v-else />
</template>

<script setup>
import { ref, onMounted } from 'vue'

const showSplash = ref(true)

// ⚠️ 这些CSS/JS需要在入口处加载一次即可
onMounted(() => {
  // splashFinished事件由ziling-splash.js内部触发
})

function onSplashFinished(e) {
  showSplash.value = false
}
</script>

<style>
/* 开屏层样式 */
.splash-layer {
  position: fixed;
  inset: 0;
  z-index: 9999;
  background: var(--zl-bg);
}
</style>
```

**方法二：在 index.html 中引入（最简单）**
```html
<!-- index.html 的 <head> 中 -->
<link rel="stylesheet" href="/splash/ziling-splash.css">
<!-- </body> 前面 -->
<script src="/splash/ziling-splash.js"></script>
```
然后在 App.vue 里放 `<div data-ziling-splash data-auto-hide="true">` 即可。

**执行要求：**
1. 在 `index.html` 中引入 `/splash/ziling-splash.css` 和 `/splash/ziling-splash.js`（方法二，最简单）
2. 在 `App.vue` 中添加 splash 挂载点 + 播放完毕后切换到 router-view
3. 不要修改 splash 包内的任何代码！它是独立交付的
4. 后期如果需要调整开屏效果（改时间/加循环等），直接编辑 `public/splash/` 下的文件即可

### 二、宠物端核心交互接口 (/ziling/api/*)

**⚠️ 这是字灵的核心功能！PetHomePage 的对话体验完全依赖这组接口。**

**鉴权说明：`/ziling/api/*` 接口无需登录即可调用（不传 Token 或不登录状态也能正常返回）。可选传 Token 以获取个性化宠物信息。与 `/api/*` 接口不同，`/ziling/api/*` 没有 JWT 鉴权中间件。**

对应文件：`src/api/ziling.js`

| 方法 | 接口 | 说明 | 对应页面/组件 |
|------|------|------|-------------|
| GET | `/ziling/api/ping` | 健康检查 | App启动时检测 |
| POST | `/ziling/api/schedule` | 日程反馈 | 任务打卡后触发 |
| POST | `/ziling/api/chat` | **核心对话** | PetHomePage（主交互） |

#### 各接口详细响应格式：

**1. chat — 核心对话响应（最重要的接口！）**
```json
{
  "quickReply": "简洁回复文本，≤20字",
  "megachar": {
    "chars": ["字"],
    "direction": "vertical",
    "rotateInterval": 0,
    "duration": 3000
  },
  "stream": [
    { "text": "一段话内容", "emoji": "^_^" },
    { "text": "另一段", "emoji": "OwO", "word": "展示的词语" }
  ]
}
```
- `quickReply`: 简短回复，显示在对话框中
- `megachar`: 汉字精灵展示数据，1~4个汉字，direction ≤2字为vertical，≥3字为horizontal
- `stream`: 3~5段消息流，每段有 text + emoji，可选 word 展示词

**chat 请求参数（请求体）：**
```json
{
  "message": "用户输入的文字",
  "history": [
    { "role": "user", "content": "上一轮用户消息" },
    { "role": "assistant", "content": "上一轮助手回复" }
  ],
  "persona": "gentle",
  "schedule": { "completed": ["任务A"], "pending": ["任务B"], "delayed": ["任务C"] }
}
```
- `message`（必填）：用户当前输入
- `history`（可选）：对话历史数组，最近 6~10 轮
- `persona`（可选）：字灵性格设定，如 `"gentle"` / `"cheerful"` / `"strict"`
- `schedule`（可选）：日程状态，打卡后传入，影响回复内容

**2. schedule — 日程反馈响应**
```json
{
  "message": "陪伴语，≤30字",
  "emoji": "^_^"
}
```

**3. validate — 组词判定响应**
```json
{
  "valid": true,
  "word": "组成的词语"
}
```
- valid=false 时无word字段
