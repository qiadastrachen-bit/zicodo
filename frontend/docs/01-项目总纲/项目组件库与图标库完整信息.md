## 3. 布局组件

### 3.1 布局组件列表

| 组件名称 | 文件路径 | 主要功能 | 使用场景 |
|---------|---------|---------|---------|
| **AppLayout** | `src/layout/AppLayout.vue` | 主应用布局，包含底部导航栏 | 登录后的主要功能页面 |
| **AuthLayout** | `src/layout/AuthLayout.vue` | 认证页面布局，适用于登录/注册 | 登录、注册等认证页面 |
| **SplashLayout** | `src/layout/SplashLayout.vue` | 启动页布局，展示开屏动画 | 应用启动、加载页面 |

### 3.2 布局组件说明

#### AppLayout（主应用布局）
- **用途**: 作为应用的主要布局框架
- **特点**: 包含底部导航栏 (ZlTabbar)，支持多个主要功能页面切换
- **适用页面**: 首页、互动、日历、个人中心等

#### AuthLayout（认证页面布局）
- **用途**: 用于登录、注册等认证流程页面
- **特点**: 简洁的布局，专注于表单操作
- **适用页面**: 登录页、注册页、认证页

#### SplashLayout（启动页布局）
- **用途**: 应用启动时的展示页面
- **特点**: 可用于展示LOGO、加载动画、初始化流程
- **适用页面**: 启动页、开屏页

## 7. 设计建议与使用指南

### 7.1 图标使用建议

1. **统一图标库**: 严格使用 `lucide-vue-next`，避免引入其他图标库
2. **图标选择原则**:
   - 选择语义明确的图标，避免模糊不清的图标
   - 参考现有代码中的图标使用方式，保持一致性
   - 访问 https://lucide.dev/icons/ 预览和选择合适的图标

3. **图标命名规范**:
   - 使用Lucide官方的图标名称（PascalCase）
   - 例如：`Home`, `Calendar`, `Settings`, `ChevronRight`

4. **ZlIcon组件使用**（推荐）:
   - 优先使用 `ZlIcon` 组件（统一封装层）
   - ZlIcon已提供统一的尺寸、颜色和交互样式
   - 便于未来图标库升级或更换时统一修改

5. **直接导入Lucide图标**（当前主要使用方式）:
   - 在需要的文件顶部导入：`import { IconName } from 'lucide-vue-next'`
   - 在模板中直接使用：`<IconName :size="20" color="#xxx" />`

### 7.2 组件开发规范

1. **新增组件原则**:
   - 功能复用性强，至少在2个以上页面使用
   - 逻辑独立，不依赖特定页面状态
   - 命名规范：以 `Zl` 开头（基础组件）或功能命名（业务组件）
   - 文件位置：放置在 `src/components/common/` 或对应功能目录下

2. **组件样式规范**:
   - 使用项目的CSS变量保持风格统一
   - 保持简洁的scoped样式，避免全局污染
   - 参考 `ZlButton.vue`, `ZlCard.vue` 等基础组件的样式结构

3. **组件Props设计**:
   - 明确的props类型定义
   - 合理的默认值
   - 清晰的事件命名和emit定义

### 7.3 设计一致性检查清单

在进行产品设计或资源制作时，请确保：

- [ ] **图标风格**: 与Lucide Icons保持一致（简约线条风格）
- [ ] **颜色系统**: 使用项目定义的CSS变量（主色、辅助色、文字色、背景色）
- [ ] **间距规范**: 保持统一的间距和留白
- [ ] **圆角规范**: 按钮、卡片、输入框的圆角保持一致
- [ ] **字体规范**: 标题、正文、辅助文字的字号和字重保持统一
- [ ] **交互反馈**: hover、active、disabled状态保持一致的交互体验
