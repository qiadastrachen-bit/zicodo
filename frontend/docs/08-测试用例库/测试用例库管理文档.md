## 一、测试用例库概述

### 1.1 目的
本测试用例库用于：
- 记录项目开发过程中的重要功能和问题事件
- 为回归测试参考
- 团队知识积累
- 持续改进依据

### 1.2 目录结构

```
docs/测试用例库/
├── 测试用例库管理文档.md          # 本文档
├── 测试用例模板.md              # 标准化模板
├── 个人中心模块/                # 功能模块目录
│   ├── 个人中心功能测试用例.md
│   ├── 用户中心-代码审查事件.md      # 今天的事件
├── 其他功能模块/               # 按功能组织
```

## 二、测试用例分类标准

### 2.1 按类型分类
| 类型 | 描述 |
|-----|------|
| **功能测试** | 验证功能是否正常 |
| **UI测试** | 界面和用户体验测试 |
| **缺陷/事件** | 记录开发过程中的重要事件 |
| **回归测试** | 修复bug后验证 |

### 2.2 按优先级分类
| 级别 | 说明 |
|-----|------|
| **P0-Critical** | 阻塞功能，必须立即处理 |
| **P1-High** | 重要功能，高优先级 |
| **P2-Medium** | 一般功能，中等优先级 |
| **P3-Low** | 优化或建议 |

## 三、测试用例创建规范

### 3.1 必须包含的内容
1. **用例ID（CaseID
2. **标题
3. **优先级
4. **前置条件
5. **测试步骤
6. **预期结果
7. **实际结果（执行时填写
8. **状态（待测试/通过/失败
9. **备注

### 3.2 命名规范
- 功能模块-功能描述-日期.md
- 示例：个人中心-代码审查事件-20260608.md
