### 问题 6：ID 生成规则不一致

**位置**：`page-prompt-add-page.md` 第 529 行、`page-prompt-team-page.md` 第 196 行、第 210 行

**问题描述**：
```js
// 个人任务 ID：时间戳
const newTask = {
  id: 'task_' + Date.now(),  // ← "task_" 前缀 + 时间戳
  // ...
}

// 团队任务 ID：不同格式
{
  id: 'team_task_1',  // ← "team_task_" 前缀 + 数字
  // ...
}

// 团队成员 ID：另一种格式
{
  id: 'm1',  // ← "m" 前缀 + 数字
  // ...
}
```

**矛盾点**：三种 ID 格式不一致，可能导致：
1. 数据类型混乱（字符串 vs 数字）
2. ID 冲突（如果不同前缀的生成规则有重叠）
3. 代码可维护性差（需要记住三种规则）

**正确做法**：
```js
// ✅ 统一 ID 生成规则：前缀 + 时间戳
const generateId = (prefix) => `${prefix}_${Date.now()}`;

// 个人任务
const newTask = {
  id: generateId('task'),
  // ...
}

// 团队
const newTeam = {
  id: generateId('team'),
  // ...
}

// 团队任务
const newTeamTask = {
  id: generateId('team_task'),
  // ...
}

// 团队成员
const newMember = {
  id: generateId('member'),
  // ...
}
```

**修复建议**：
1. 在 `src/utils/id.js` 中创建统一的 ID 生成函数
2. 修改 `schedule.js` 和 `teams.js` 中的 ID 生成逻辑
3. 更新所有提示词文件中的示例代码

**预计修复时间**：1 小时（需要修改 3 个 Store 文件 + 2 个提示词文件）

  // ...

// 团队任务
{
  id: 'team_task_1717824000000',
  title: 'can you send us the template please?',
  date: '2026-06-21',
  startTime: '08:30',
  endTime: '17:30',
  note: '',
  location: '',
  assignee: '页面',  // ← 团队任务才有这个值
  fromTeam: 'team_1717824000000',  // ← 团队 ID
  // ...
