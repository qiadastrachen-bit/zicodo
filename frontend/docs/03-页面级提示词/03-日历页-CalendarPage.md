## ✅ 已确认决策（不要再问用户）

### 决策1：进入日历页时的默认选中行为
**✅ 决定：选项A（推荐）**
- **如果今天有任务**：默认选中今天
- **如果今天没有任务**：不选中任何日期，待办区显示空状态

**理由**：
- 用户最关心今天的任务
- 如果今天没任务，页面看起来"空"是正常的

---

### 决策2：月份切换后的选中状态
**✅ 决定：选项A（推荐）**
- **切换月份后**：重置为不选中任何日期
- **如果新月份有"今天"（比如当月）**：选中今天
- **如果切换到其他月份**：不选中

**理由**：
- 避免混淆（用户可能忘记自己选中的是哪个日期）
- 符合移动端交互习惯

---

### 决策3：日期小圆点的显示规则
**✅ 决定：3.1 选A；3.2 选A**
- **3.1**：只显示 **1 个点**（不区分数量，仅表示"有任务"）
- **3.2**：全部任务完成后，**不显示点**（或显示灰色点，表示已完成）

**理由**：
- 1个点足够表示"有任务"，不需要显示数量
- 任务完成后，日期下方不应该有干扰性的标记

---

### 决策4：待办区空状态
**✅ 决定：选项A（推荐）**
- **显示空状态组件**（使用 `EmptyState.vue`，显示 `loading.png` + "今天没有任务"）

**理由**：
- 符合设计规范，统一空状态处理
- 用户明确知道"今天没有任务"，而不是"页面出错了"

---

### 决策5：任务卡片的交互
**✅ 决定：5.1 选A；5.2 选A；5.3 选B**
- **5.1**：点击复选框 → **直接标记完成/未完成**，不跳转页面
- **5.2**：点击任务名称/卡片（非复选框区域） → **跳转到任务详情页**
- **5.3**：日历页**可以编辑**（点击后进入编辑模式，复用 + 页面的编辑弹窗）

**理由**：
- 5.1：快速标记完成，不需要跳转
- 5.2：查看任务详情，需要跳转
- 5.3：日历页应该支持快速编辑，不需要跳转到 + 页面

---

### 决策6：任务详情页的功能
**✅ 决定：6.1 选B；6.2 选B；6.3 选A**
- **6.1**：详情页**可编辑**，有"编辑"按钮（点击后打开编辑弹窗）
- **6.2**：详情页**有"删除"按钮**
- **6.3**：点击返回按钮 → **回到日历页**（保留之前的选中状态）

**理由**：
- 6.1：用户可能需要快速编辑任务，不需要返回日历页再编辑
- 6.2：删除功能应该在详情页也能访问
- 6.3：返回时保留选中状态，用户不需要重新选择日期

---

### 决策7：与 + 页面（任务页）的数据同步
**✅ 决定：7.1 选A；7.2 选A**
- **7.1**：**完全同步**，共用同一个 `scheduleStore`（localStorage）
- **7.2**：日历页**没有"添加任务"按钮**，添加任务去 + 页面

**理由**：
- 7.1：数据一致性，避免冲突
- 7.2：添加任务的功能在 + 页面已经存在，不需要重复

### 决策8：今天的特殊标记

## 🎯 功能需求

### 1. 月份导航

**位置**：页面顶部（48px 高度）

**元素**：
- **左侧**：`<` 按钮（切换到上个月）
- **中间**：`YYYY年MM月`（如 `2026年6月`）
- **右侧**：`>` 按钮（切换到下个月）

**交互**：
- 点击 `<` → 切换到上个月
- 点击 `>` → 切换到下个月
- **切换后**：重置选中状态（不选中任何日期）
  - 如果新月份有"今天"（比如当月），选中今天
  - 如果切换到其他月份，不选中

**手势**：
- **左滑**：切换到下个月
- **右滑**：切换到上个月

---

### 2. 日历网格

**星期标题**（32px 高度）：
```
M  T  W  T  F  S  S
```
- 周起始日：**周一**（符合中国习惯）
- 字号：14px（--zl-font-sm）
- 颜色：--zl-text-secondary（次要文字）

**日期网格**：
- 每个日期格子：**40×40px**（正方形，方便点击）
- 日期数字：16px（--zl-font-base）
- 选中状态：绿色圆圈背景（--zl-brand: #87C8B4）
- 今天标记：**绿色数字**（无论是否选中）

**非当月日期**：
- 灰色淡化显示（opacity: 0.3 或 color: var(--zl-text-tertiary)）

**有任务的日期**：
- 日期数字下方显示 **1 个小圆点**（4×4px，--zl-brand 颜色）
- 全部任务完成后，**不显示点**

---

### 3. 待办列表

**位置**：日历网格下方

**标题**：
- 文字：`今日任务`（如果选中了日期）
- 文字：`请选择日期`（如果没有选中日期）
- 字号：20px（--zl-font-md）
- 字重：500（--zl-weight-medium）

**任务卡片**（48px 高度）：
- **左侧**：复选框（☑️ 完成 / ☐ 未完成）
- **标题**：任务名称（如 `跑步锻炼`）
- **时间**：开始时间 或 时间范围（如 `7:20 AM` 或 `18:40PM-19:00PM`）
- **🕐 图标**：时间图标（装饰）
- **✓ 图标**：进行中的任务显示编辑/打卡图标

**点击行为**：
- 点击复选框 → 切换完成状态（不跳转页面）
- 点击卡片区域（非复选框） → 跳转到任务详情页

**空状态**：
- 如果没有任务，显示 `EmptyState.vue` 组件
- 图片：`/loading.png`
- 文案：`今天没有任务` 或 `请选择日期查看任务`

---

### 4. 任务详情页

**触发**：点击任务卡片（非复选框区域）

**路由**：`/tasks/:id`（已在 src/router/index.js 第94-98行定义）

**内容**：
- **顶部栏**：
  - 左侧：返回按钮（←）→ 回到日历页（保留之前的选中状态）
  - 中间：任务标题（如 `跑步锻炼`）
  - 右侧：删除按钮（文字）→ 删除任务

- **日期**：`Calendar 图标 21 Jun 2026`
- **时间范围**：`Clock 图标 8:30 AM — 5:30 AM`
- **备注**：`FileText 图标 记得带水杯`
- **地点**：`MapPin 图标 球场`

- **底部按钮**：
  - `编辑` → 打开编辑弹窗（复用 + 页面的编辑弹窗）

---

### 5. 编辑弹窗（复用 + 页面）

**触发**：在任务详情页点击"编辑"按钮

**表单字段**（与 + 页面相同，但预填充数据）：

| 字段 | 图标 | 类型 | 示例 |
|-------|------|------|------|
| 日期范围 | Calendar | 日期选择器 | 4月1日-5月1日（每天）|
| 开始时间 | Clock | 时间选择器 | 19:30 PM |
| 结束时间 | Clock | 时间选择器 | 20:30 PM |
| 备注 | FileText | 文本输入 | 记得带水杯 |
| 地点 | MapPin | 文本输入 | 操场 |
| 闹铃 | Bell | Toggle 开关 | 开/关 |

**按钮**：
- "确认" → 保存修改 → 关闭弹窗 → 返回任务详情页

**额外操作**：
- 右上角"删除"文字按钮 → 删除任务
- 右上角复选框 → 标记完成/未完成

## 🗄️ 数据模型（与 + 页面同步）

### Task 对象（localStorage 存储）

```js
{
  id: 'task_1717824000000',        // 唯一 ID（时间戳）
  title: '跑步锻炼',               // 任务标题
  date: '2026-06-21',              // 任务日期（YYYY-MM-DD）
  dateRange: {                      // 日期范围（可选）
    start: '2026-04-01',
    end: '2026-05-01'
  },
  recurrence: '每天',               // 重复规则：'每天' | '每周X' | null
  startTime: '19:30',              // 开始时间（HH:mm）
  endTime: '20:30',                // 结束时间（HH:mm，可选）
  note: '记得带水杯',               // 备注
  location: '操场',                // 地点
  alarmEnabled: false,             // 闹铃开关
  completed: false,                // 是否完成
  checkInStart: null,              // 打卡开始时间（ISO 字符串）
  checkInEnd: null,                // 打卡结束时间（ISO 字符串）
  type: 'personal',                // 'personal' | 'team'（MVP 仅 personal）
  createdAt: '2026-06-08T10:00:00Z',
  updatedAt: '2026-06-08T10:00:00Z'
}
```

### localStorage Key

```js
const STORAGE_KEY = 'zilin_schedule_tasks'
```

### Pinia Store（复用 + 页面的 store）

**文件路径**：`src/stores/schedule.js`

**说明**：日历页面**不直接访问 localStorage**，而是通过 `scheduleStore` 来读取和写入数据。

**引入方式**：
```js
import { useScheduleStore } from '@/stores/schedule'

const scheduleStore = useScheduleStore()
const tasks = computed(() => scheduleStore.tasks)
```
