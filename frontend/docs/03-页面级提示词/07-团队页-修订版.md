2. **创建 `JoinTeamForm.vue` 组件**（加入团队弹窗）：

   ```vue
   <template>
     <div class="join-overlay" @click.self="$emit('close')">
       <div class="join-modal">
         <div class="modal-header">
           <button class="close-btn" @click="$emit('close')">×</button>

         </div>

           <input 
             type="text"

           />

             @click="handleJoin"

     if (team) {
       store.joinTeam(team.id)
       emit('join')
       emit('close')
     } else {
       alert('邀请码无效')
     }
   }
   </script>

3. **修改 `teams.js` Store，添加 `findTeamByInviteCode` 和 `joinTeam` 函数**：

   ```js
   function findTeamByInviteCode(code) {
     return teams.value.find(t => t.inviteCode === code)
   }
   
   function joinTeam(teamId) {
     const team = teams.value.find(t => t.id === teamId)
     if (team) {
       // 添加当前用户到成员列表
       const currentUser = getCurrentUser()
       team.members.push({
         id: 'm_' + Date.now(),
         name: currentUser,
         role: '成员'
       })
       saveToStorage(teams.value)
     }
   }
   ```

### 修订3：团队创建时自动生成邀请码

**目标**：团队创建时自动生成6位数字邀请码。

**实现方式**：

修改 `teams.js` 的 `createTeam` 函数：

```js
function createTeam(formData) {
  // 生成6位数字邀请码
  const inviteCode = Math.floor(100000 + Math.random() * 900000).toString()
  
  const newTeam = {
    id: 'team_' + Date.now(),
    ...formData,
    inviteCode: inviteCode,  // ← 添加邀请码
    members: formData.members || [
      { id: 'm1', name: getCurrentUser(), role: '组长' }
    ],
    tasks: [],
    createdAt: new Date().toISOString()
  }
  teams.value.unshift(newTeam)
  saveToStorage(teams.value)
  return newTeam
}
```

### 修订4：设计规范统一性

## 设计规范统一性（强制）

### 必须使用 Design Tokens

所有样式必须使用 `tokens.css` 中定义的 CSS 变量，禁止硬编码颜色/尺寸/间距。

**错误示例**：

  border-radius: 12px;  /* ❌ 硬编码 */
}
```

### 细节处理要求

1. **过渡动画**：所有可交互元素必须有 `transition`（使用 `var(--zl-transition-fast)`）
2. **按下反馈**：所有可点击元素必须有 `:active { transform: scale(0.98) }`（或类似反馈）
3. **禁用状态**：按钮/输入框禁用时，必须有视觉反馈（`opacity: 0.5; cursor: not-allowed`）
4. **加载状态**：所有异步操作必须有加载状态（使用 `LoadingSpinner` 组件）
5. **错误状态**：所有错误必须有用户友好的提示（使用 `ErrorMessage` 组件）
6. **空状态**：所有空列表必须有空状态提示（使用 `EmptyState` 组件）
7. **响应式**：所有页面必须适配 `375px ~ 428px` 宽度（主流手机屏幕）

### 组件复用要求

尽可能复用已有组件，避免重复造轮子：

- 任务卡片 → 复用 `TaskCard.vue`
- 弹窗 → 复用 `TaskForm.vue` 的弹窗逻辑
- 空状态 → 使用 `LoadingState.vue` 组件（加载素材图）
- 错误状态 → 使用 `ErrorState.vue` 组件（错误素材图）

#### 轮播文案功能（新增）

> ⚠️ **文案可以自定义、可以轮播、自由变换**！

**实现方式**：

1. **修改 `LoadingState.vue`，支持轮播文案**：
   ```vue
   <template>
     <div class="loading-state">
       <img src="/loading.png" alt="加载中" class="loading-img" />
       <p class="loading-text">{{ currentMessage }}</p>
     </div>
   </template>

   <script setup>
   const props = defineProps({
     messages: { 
       type: Array, 
       default: () => ['加载中...']  // ← 支持多个文案
     },
     interval: { 
       type: Number, 
       default: 3000  // ← 轮播间隔（毫秒）
     }
   })

   const currentIndex = ref(0)
   const currentMessage = computed(() => props.messages[currentIndex.value])

   let timer = null

   onMounted(() => {
     if (props.messages.length > 1) {
       timer = setInterval(() => {
         currentIndex.value = (currentIndex.value + 1) % props.messages.length
       }, props.interval)
     }
   })

   onUnmounted(() => {
     if (timer) clearInterval(timer)
   })
   </script>
   ```

2. **使用示例**：
   - **单个文案**（不轮播）：
     ```vue
     <LoadingState message="加载中..." />
     ```
   
   - **多个文案**（自动轮播）：
     ```vue
     <LoadingState 
       :messages="['功能正在上线', '敬请期待', '即将开放']" 
       :interval="3000" 
     />
     ```
   
   - **自由变换**（动态修改）：
     ```vue
     <LoadingState :messages="dynamicMessages" />
     ```
     ```js
     // 可以随时修改 dynamicMessages，文案会自动更新
     const dynamicMessages = ref(['功能正在上线', '敬请期待'])
     setTimeout(() => {
       dynamicMessages.value = ['加载中...']  // ← 自由变换
     }, 5000)
     ```

**默认轮播文案**（功能正在上线/正在设计）：
